# 過去セッション記憶の検索

agent-memory プラグインのハイブリッド検索エンジンで過去の会話を検索する。
キーワード(FTS5)とセマンティック(Vec)の両方でマッチングし、
関連度×鮮度の2因子でランキングする。

## 検索コマンド

```bash
cd ~/.claude/plugins/local/agent-memory/scripts
.venv/Scripts/python search.py "<検索クエリ>"
```

### プロジェクト絞り込み

```bash
.venv/Scripts/python search.py "project:<ID> <クエリ>"
```

利用可能なプロジェクトIDは `config/settings.toml` の `[projects]` セクションを参照。
現在のcwdから自動推定もできるが、明示指定の方が精度が高い。

## 検索戦略

ユーザーの質問が曖昧な場合、以下の戦略で検索精度を上げる:

1. **同義語展開**: ユーザーの言葉から複数のキーワードを考え、最も具体的なものから試す
   - 例: 「費用」→ まず「ランニングコスト 月額」、次に「費用 見積もり」
2. **プロジェクト絞り込み**: 文脈からプロジェクトが特定できるなら `project:` を使う
3. **リトライ**: 結果が0件や不十分なら、別の言い換えで再検索する
4. **複数検索の組み合わせ**: 異なる角度から2-3回検索し、結果を統合する

## 結果の扱い

- 検索結果をそのまま貼り付けない。**要点を整理して回答する**
- 日付・数値・人名は検索結果から正確に引用する
- 複数セッションにまたがる情報は時系列で整理する
- 結果に矛盾がある場合は最新のものを優先し、矛盾を明示する

## 検索方式（参考）

- **FTS5**: fugashi 分かち書き + BM25（3段階フォールバック: OR → prefix → LIKE）
- **Vec**: ruri-v3-30m cosine KNN
- **RRF(k=60)** でマージ → **2因子リランキング**（relevance 0.7 × recency 0.3）

## Auto Memoryとの使い分け

- **Auto Memory** (`~/.claude/projects/*/memory/`): 要約済みの安定情報。セッション開始時に自動ロード
- **このスキル**: 生の会話ペアを全文検索。Auto Memoryにない詳細・文脈・経緯を掘り起こす

Auto Memoryで見つからない、または「正確な発言内容」「議論の経緯」が必要な場合にこのスキルを使う。
